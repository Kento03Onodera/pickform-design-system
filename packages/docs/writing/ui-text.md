# UIテキスト

アプリケーション上の表現で迷ったときの判断基準や、推奨する表記を記載しています。

---

## 括弧「 」、［ ］、【 】の使い分け

[ライティングスタイル「括弧類は「 」、［ ］、【 】、（ ）を使用する」](https://smarthr.design/products/contents/idiomatic-usage/symbol/)に準拠して、全角記号を使用します。

アプリケーション上での括弧の使い分けは以下です。ヘルプページやリリースノートも準じます。

### 文中でファイル名を示すときに囲うもの

- ファイル名：`「 」` — 地の文と区別したい、強調したいとき

### 画面上の表示を示すときに囲うもの

| 対象 | 使用する括弧 |
| --- | --- |
| 画面名、欄、操作ボタン名 | `［ ］欄`、`［ ］ボタン` |
| 設定値（プルダウンの選択肢なども含む） | `「 」` |
| カスタムオブジェクト名 | `【 】` |

`【 】`は、以下のような場合に適宜使いましょう。

- 注意して操作してほしいとき
- 強調させたいとき
- ダイアログなどで操作手順を示すとき

## 日付と時間の表記

日付（date）の文字列形式は、他社アプリケーションと連携しやすいよう、「YYYY/MM/DD」で表記します。

- YYYY：西暦年4桁
- MM：2桁の月
- DD：2桁の日

区切りには「/」を使用します。

時刻（time）の文字列形式は、HH:MMで表記します。

- HH：24時間制の2桁
- MM：2桁の分

区切りには「:」を使用します。

## 助詞の使い分け

### アクションに関する画面やダイアログのタイトルでは、助詞は「の」を使用する

操作画面やアクションダイアログで、オブジェクトを明記しながら操作を表す場合には、助詞に「の」を使用します。[削除ダイアログ](https://smarthr.design/products/design-patterns/delete-dialog/)などの一部を除き、タイトルには名前を示したいので、操作を動名詞として扱う「の」を伴います。

> **OK**: 従業員の追加
> **NG**: 従業員を追加

### ボタンやテキストリンクでは、助詞は「を」を使用する

ボタンやテキストリンクで、オブジェクトを明記しながら操作を表す場合には、助詞に「を」を使用します。ボタンやテキストリンクはアクションを促すものなので、直接的な目的語を示す「を」を伴います。

> **OK**: 従業員を追加
> **NG**: 従業員の追加

### 画面タイトル、項目名、ボタンは名詞、動名詞で表記する

画面タイトル、項目名、ボタンのラベルは名詞や動名詞で表記します。

### 移動の操作を表す時の助詞は、具体的な対象なら「に」、相対的な位置なら「へ」

具体的な画面やオブジェクトへの移動は「に」、相対的な位置や方向を示す場合は「へ」を使い分けます。

## テキストリンク・ボタン

### テキストリンクは「こちら」ではなく、移動先を想起しやすい表現をする

テキストリンクは「こちら」ではなく、実際の移動先を想起できる具体的な表現を使用します。

### 他ページへのアクセスは「"ページ名"ページへ」と表現

他ページへのアクセスを案内する場合は、「"ページ名"ページへ」と記載します。

---

## テーブル・一覧

### よくあるテーブルの見出しに「一覧」はつけない

テーブルの見出しに不要な「一覧」は付けません。

## 作成・登録・追加の使い分け

### 作成を表す言葉（登録・追加・作成の考え方）

[SmartHR Design System 参考](https://smarthr.design/products/contents/app-writing/#rec9RG6ZyvjvAVwli-0)

状況に応じて「登録」「追加」「作成」を使い分けます。

| 表現 | 意味・用途 |
| --- | --- |
| **作成** | 新しいオブジェクトを生み出す操作全般 |
| **追加** | 既存のリストやグループに項目を増やす操作 |
| **登録** | データベースや管理上の台帳に正式に記録する操作 |

---

## 操作を止めるとき

### 操作を止めるときの言葉（キャンセル・取り消し・中断の考え方）

状況に応じて「キャンセル」「取り消し」「中断」を使い分けます。

| 表現 | 意味・用途 |
| --- | --- |
| **キャンセル** | 操作自体を取りやめること（ダイアログの閉じるなど） |
| **取り消し** | 実行した操作を元に戻すこと |
| **中断** | 途中で止めること（再開の可能性がある） |

---

## 状態を戻すとき

### 状態を戻すときの言葉（解除・差し戻し・戻すの考え方）

状況に応じて「解除」「差し戻し」「戻す」を使い分けます。

| 表現 | 意味・用途 |
| --- | --- |
| **解除** | 設定や制限を取り除くこと |
| **差し戻し** | ワークフロー上で前の工程に戻すこと |
| **戻す** | 一般的に元の状態に戻すこと |

---

## ファイルの取り込み

### ファイルを取り込んで操作するときの文言

ファイルのインポート操作に関する文言は、以下の形式で統一します。

- アップロード時：「ファイルをアップロード」
- インポート時：「ファイルをインポート」

> [!NOTE]
> 「アップロード」はファイルをサーバーに転送する操作、「インポート」はデータをアプリケーションに適用する操作です。用途に応じて使い分けてください。

---

## バックグラウンド処理

### バックグラウンド処理をリクエスト時の文言は「{処理名}を受け付けました」

バックグラウンド処理を開始する際のメッセージは、「{処理名}を受け付けました」の形式で統一します。

> **OK**: エクスポートを受け付けました
> **NG**: エクスポートを開始しました
